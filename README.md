# Neovim Конфиг

Конфигурация Neovim для разработчиков, перешедших с VS Code. Использует `vim-plug` в качестве менеджера плагинов.

## Требования

- **Neovim 0.9+**
- **Git**
- **cURL** (для Kimi Usage и автозагрузки vim-plug)
- **Build tools**: `make`, `gcc`, `g++` (для telescope-fzf-native и Treesitter)
- **ripgrep** (`rg`) — поиск по тексту в Telescope
- **fd** (`fd-find`) — быстрый поиск файлов в Telescope
- **Node.js + npm** — Treesitter парсеры и LSP через Mason
- **Python 3 + pip** — LSP (pyright) и форматтер (black)
- **[Crush CLI](https://github.com/charmbracelet/crush)** — опционально, для Codock

### Опционально (языковые серверы)

- **Go** — для `gopls`
- **Rust (rustup)** — для `rust_analyzer`

### Поддерживаемые дистрибутивы

Скрипт `install.sh` автоматически устанавливает зависимости на:

| Дистрибутив | Пакетный менеджер |
|-------------|-------------------|
| Ubuntu, Debian, Pop!_OS | `apt` |
| Arch Linux, Manjaro, EndeavourOS, CachyOS | `pacman` |

На других системах зависимости нужно установить вручную.

## Установка

### 1. Клонирование

Клонируйте репозиторий в удобное место (не в `~/.config/nvim`):

```bash
git clone https://github.com/Persik-V-python/nvim-dotfiles.git ~/nvim-dotfiles
cd ~/nvim-dotfiles
```

### 2. Запуск установки

Скрипт создаст символическую ссылку `~/.config/nvim` → папка `nvim/` в репозитории:

```bash
./install.sh
```

Существующий конфиг будет сохранён с суффиксом `.bak.<timestamp>`.

### 3. Установка плагинов

При первом запуске Neovim автоматически загрузит `vim-plug`. Затем выполните:

```vim
:PlugInstall
```

После установки перезапустите Neovim.

### 4. Установка LSP-серверов

```vim
:Mason
```

Откроется интерфейс для установки языковых серверов (LSP), линтеров и форматтеров.

### 5. Установка парсеров Treesitter

```vim
:TSUpdate
```

### 6. Установка FZF-расширения для Telescope

Если `telescope-fzf-native` не собрался автоматически:

```bash
cd ~/.local/share/nvim/plugged/telescope-fzf-native.nvim
make
```

### 7. Настройка Kimi API токена (опционально)

Для мониторинга лимитов Kimi Usage добавьте API-токен **одним** из способов:

**Способ 1 — переменная окружения (рекомендуется):**
```bash
# ~/.bashrc, ~/.zshrc или ~/.profile
export KIMI_API_TOKEN="your-token-here"
```

**Способ 2 — глобальная переменная в init.vim:**
```vim
" ~/.config/nvim/init.vim
let g:kimi_api_token = 'your-token-here'
```

После настройки перезапустите Neovim — в правом нижнем углу появится окно с прогресс-барами.

## Kimi Usage — мониторинг лимитов

Встроенный плагин, который каждые **2 минуты** опрашивает API Kimi и отображает прогресс-бары в плавающем окне:

- **Недельный лимит** — общее использование
- **5-минутный лимит** — окно 300 секунд
- **Квота** — оставшаяся квота

### Команды

| Команда | Описание |
|---------|----------|
| `:KimiUsage` | Показать/скрыть окно |
| `:KimiUsageStart` | Запустить мониторинг |
| `:KimiUsageStop` | Остановить мониторинг |
| `:KimiUsageFetch` | Обновить данные вручную |

### Горячая клавиша

| Клавиша | Действие |
|---------|----------|
| `<Пробел>ku` | Показать/скрыть Kimi Usage |

## Быстрый старт

### Основные команды

| Команда | Описание |
|---------|----------|
| `:PlugInstall` | Установить все плагины |
| `:PlugUpdate` | Обновить плагины |
| `:PlugClean` | Удалить неиспользуемые плагины |
| `:Mason` | Управление LSP-серверами |
| `:TSUpdate` | Обновить парсеры Treesitter |
| `:checkhealth` | Проверить состояние системы |

### Важные горячие клавиши (Leader = `Пробел`)

| Клавиша | Действие |
|---------|----------|
| `<Пробел>e` | Открыть/закрыть проводник файлов |
| `<Пробел>ff` | Найти файл |
| `<Пробел>fg` | Поиск по тексту |
| `<Пробел>bn` | Следующий буфер |
| `<Пробел>bp` | Предыдущий буфер |
| `<Пробел>CC` | Открыть Crush (AI CLI) |
| `gd` | Перейти к определению |
| `K` | Показать документацию |
| `<Пробел>rn` | Переименовать символ |

## Структура

```
nvim/
├── init.vim              # Точка входа, vim-plug
├── autoload/
│   └── plug.vim          # vim-plug
└── lua/
    ├── core/
    │   ├── options.lua   # Основные настройки
    │   ├── keymaps.lua   # Горячие клавиши
    │   └── autocmds.lua  # Автокоманды
    └── plugins/
        ├── theme.lua
        ├── ui.lua
        ├── finder.lua
        ├── treesitter.lua
        ├── editor.lua
        ├── lsp.lua
        ├── git.lua
        ├── codock.lua
        ├── persistence.lua
        └── kimi_usage.lua
```
